# FES READY（フェス レディ）

## 概要

**FES READY** は、音楽フェス参加者向けに  
**「予習・準備・当日の行動」** を一元管理できる Web アプリです。

音楽フェスに行く際、

- タイムテーブルは公式アプリ  
- 予習は音楽サブスク  
- 持ち物はメモアプリ  
- 天気は天気アプリ  

と、必要な情報が複数のサービスに分散している課題があります。

FES READY はこれらを一つにまとめ、  
**フェス初心者でも迷わず準備でき、当日を最大限楽しめる体験**を提供します。

本アプリは、**個人開発として要件定義・設計・実装・運用まで一貫して行いました。**

## サービスURL

https://fesready.com

## 開発背景・作成理由

初めて音楽フェスに参加した際、  
知らなかったアーティストの楽曲に出会い、  
「音楽の世界が一気に広がる体験」をしました。

一方で、初参加者にとっては以下のような不安も多く感じました。

- 何を持って行けばいいのかわからない  
- 天候や気温への不安  
- どのアーティストの、どの曲を聴いておくべきかわからない  

これらの不安を解消できれば、  
**フェスはもっと気軽で、もっと楽しい体験になる**と考え、本サービスを開発しました。

## 想定ユーザー

- 音楽フェスに初めて参加する人  
- 複数のアプリで情報を管理するのが面倒な人  
- 効率よくフェスの予習をしたい人  

## 主な機能

### 共通機能（未ログイン）

- フェス情報の検索・閲覧  
  - 日程 / 会場 / 出演アーティスト / タイムテーブル
- アーティスト情報の閲覧（Spotify連携）
- 過去フェスのセットリスト閲覧
- フェス・アーティストごとの予習リスト閲覧・試聴

### ログインユーザー向け機能

- マイフェス管理（参加予定フェスの保存）
- お気に入りアーティスト管理
- **マイタイムテーブル作成**
  - 当日の行動を可視化
  - SNSでのシェア機能
- 持ち物チェックリスト作成・編集
- 天気連携（Open-Meteo）
  - 日別・時間別の気温 / 降水量表示
- マイページ編集（ニックネーム等）

### 管理者機能

- フェス / アーティスト / 出演枠情報の管理
- 楽曲・セットリスト情報の管理
- 持ち物リストテンプレート管理

※ セットリスト情報は正確性が重要なため、  
ユーザー投稿ではなく管理者入力としています。

## 設計・実装で意識した点

### 1. 初心者でも迷わない導線設計
- 「予習 → 当日 → 準備」が自然につながる画面構成
- 初回利用時でも直感的に操作できるUIを意識

### 2. 再利用性を意識したデータ設計
- フェス / アーティスト / セットリストを正規化
- 複数フェス・複数年開催を想定したスキーマ設計
- お気に入り・マイタイムテーブルは中間テーブルで管理

### 3. 外部APIに依存しすぎない構成
- Spotify / 天気APIは参照用途に限定
- コアデータはDBに保持し、API障害時も最低限利用可能

## 今後の展望

- 予習曲のプレイリスト化
  - 予習ページで選択した楽曲をまとめてプレイリスト化し、
    Spotifyで連続再生できるようにすることで、移動中や作業中でも効率的に予習可能にする

- レコメンド機能の改善  
  - ユーザーのお気に入り登録やマイタイムテーブルの情報をもとに、
    フェス当日に楽しめる可能性の高い楽曲・アーティストを提案する仕組みを検討

- PWA対応  
  - フェス会場での電波状況を考慮し、
    タイムテーブルや持ち物リストなどの基本情報をオフラインでも確認可能にする

## まとめ

FES READY は、  
**「フェスに行く前から、もう楽しい」** 体験を目指して開発したアプリです。

フェス初心者の不安を解消し、  
音楽との新しい出会いを後押しすることを目的としています。
